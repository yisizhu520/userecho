# 聚类算法对比报告

**数据集**: 48 条反馈
**Embedding Provider**: VOLCENGINE

---

## 📊 总体对比

| 方案 | 聚类数 | 已聚类 | 噪声点 | 聚类率 | 说明 |
|------|--------|--------|--------|--------|------|
| **DBSCAN** | 4 | 18 | 30 | 37.5% | 旧方案，固定阈值 |
| **HDBSCAN** | 4 | 26 | 22 | 54.2% | 方案1，自适应阈值 |
| **HDBSCAN+LLM** | 7 | 21 | 27 | 43.8% | 方案2，语义校验 |

**指标说明**:
- **聚类数**: 形成的聚类数量，越少说明归纳能力越强
- **已聚类**: 成功聚类的反馈数量
- **噪声点**: 无法归类的反馈（孤立点）
- **聚类率**: 已聚类反馈占总数的百分比

## ⚙️ 参数配置

### DBSCAN

```python
algorithm = dbscan
threshold = 0.9
min_samples = 3
```

### HDBSCAN

```python
algorithm = hdbscan
min_cluster_size = 3
min_samples = 2
```

### HDBSCAN+LLM

```python
algorithm = hdbscan+llm
min_cluster_size = 3
min_samples = 2
llm_min_size = 3
validation_count = 4
split_count = 3
sub_cluster_count = 6
```

## 🎯 结论与建议

### 质量分析

请人工检查以下方面：

1. **聚类准确性**: 查看各方案的聚类详情，检查是否有不相关的反馈被错误聚在一起
2. **聚类完整性**: 检查是否有明显相似的反馈没有被聚在一起
3. **噪声合理性**: 检查噪声点是否确实难以归类
4. **聚类粒度**: 聚类是否太细（过拟合）或太粗（欠拟合）

### 方案对比

- **DBSCAN vs HDBSCAN**: 聚类数变化 +0.0%
- **HDBSCAN vs HDBSCAN+LLM**: 聚类数变化 +75.0%（LLM 拆分了 3 个聚类）

### 配置建议

根据对比结果，建议调整配置：

```bash
# server/backend/.env

# 选择聚类算法（true=HDBSCAN, false=DBSCAN）
CLUSTERING_USE_HDBSCAN=true

# HDBSCAN 参数
CLUSTERING_HDBSCAN_MIN_CLUSTER_SIZE=3  # 最小聚类大小
CLUSTERING_HDBSCAN_MIN_SAMPLES=2        # 最小样本数

# LLM 校验（可选，会增加成本）
CLUSTERING_LLM_VALIDATION_ENABLED=false  # 是否启用
CLUSTERING_LLM_VALIDATION_MIN_SIZE=5     # 多大的聚类需要校验
```
