# 小红书高级软文创作工作流

本 Workflow 用于生成**去广告化、价值驱动、高审美**的小红书图文内容。

## 1. 选题定位 (Value Definition)

首先明确文章的核心价值角度，从以下三个方向中选择其一：

-   **认知突围型 (Insight)**: 颠覆用户固有认知，指出"伪需求"或"盲区"。
    -   *示例*: "90%的产品经理都在做搬运工"
-   **职业进阶型 (Methodology)**: 分享高阶工作流或SOP，工具作为落地手段。
    -   *示例*: "拒绝做人肉路由器：高级PM的自动化闭环"
-   **独立开发故事型 (Story)**: 个人成长经历、杠杆思维、一人企业心法。
    -   *示例*: "一人公司生存指南：如何建立虚拟团队"

## 2. 内容撰写 (Content Drafting)

遵循 **Golden Circle (Why - How - What)** 结构撰写文案：

1.  **Why (痛点/共鸣)**: 描述现状的痛，指出问题的本质。占篇幅 40%。
    -   *关键*: 制造焦虑或共鸣，让用户觉得"这就是我"。
2.  **How (方法论/干货)**: 提供通用的解决思路或SOP。占篇幅 50%。
    -   *关键*: 即使不用我们的产品，这套方法论也有价值。
3.  **What (工具落地)**: 自然引出 UserEcho 作为高效落地的工具。占篇幅 10%。
    -   *关键*: "为了落地这套方法，我用了这个工具..."

**风格要求**:
-   去广告味：禁止"立即购买"、"最好用的工具"等硬广词汇。
-   专业感：多用"工作流"、"闭环"、"杠杆"、"颗粒度"等行业黑话（适度）。
-   排版：多留白，多用 emoji 做列表符。

## 3. 视觉设计 (Visual Strategy)

生成 2-3 张高配合度的图片。

### 3.1 封面图 (Cover Image)
使用 **视觉隐喻** 或 **高对比场景**，避免直接放产品截图。
-   **Prompt 关键词**: `Minimalist`, `Concept Art`, `Metaphor`, `Iceberg`, `Split screen`, `Chaos vs Order`.
-   **比例**: 3:4 (Vertical).

### 3.2 内容图 (Content Images)
-   **实拍感场景图**: MacBook/iPad 屏幕特写 + 咖啡/笔记本，展现"掌控感"。
-   **Notion风卡片**: 极简排版，清单体，流程图。
-   **Prompt 关键词**: `Lifestyle photography`, `Macbook screen`, `Depth of field`, `Clean UI`, `Notion style layout`.

## 4. 交付检查 (Review Checklist)

- [ ] 标题是否足够吸睛（除了产品名，是否有痛点/利益点）？
- [ ] 封面图是否有高级感（拒绝廉价拼接）？
- [ ] 正文是否提供了非产品的"干货价值"？
- [ ] 产品植入是否自然（作为Solution的一部分）？
- [ ] 互动引导是否克制（引导私信/文档，而非直接甩链接）？

---

**执行建议**:
在生成图片时，优先尝试生成"实拍感"和"概念插画"两种风格供用户选择。
