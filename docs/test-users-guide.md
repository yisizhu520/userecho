## 🚀 快速创建测试用户

### 方法 1：使用 SQL 脚本（推荐）

```bash
# 进入后端目录
cd server/backend

# 执行 PostgreSQL 脚本
psql -U your_username -d your_database -f sql/postgresql/init_test_users.sql
```

**说明：**
- 脚本会自动创建角色（如果不存在）
- 脚本会自动创建用户（如果不存在）
- 脚本会跳过已存在的用户
- 执行后会显示创建结果

### 方法 2：使用 Python 脚本（如果依赖已安装）

```bash
# 进入后端目录
cd server/backend

# 执行 Python 脚本
python scripts/create_test_users.py
```

**前提条件：**
- 需要安装项目依赖：`pip install -r requirements.txt`
- 需要配置数据库连接

## ⚠️ 注意事项

### 1. 角色需要关联菜单

创建用户后，需要确保角色已经关联了对应的菜单。

**检查方法：**
1. 使用 `admin` 账号登录
2. 进入"系统管理 > 角色管理"
3. 点击"修改"按钮，查看每个角色关联的菜单
4. 如果菜单为空，需要手动勾选相应的菜单

**业务角色菜单分配建议：**
- **PM**：勾选"反馈管理"目录下的全部菜单
- **CS**：勾选"反馈列表"和"客户管理"
- **开发**：勾选"需求主题"（只读）
- **老板**：勾选"反馈管理"目录下的全部菜单

**系统角色菜单分配建议：**
- **系统管理员**：勾选"系统管理"、"日志"、"监控"等系统菜单

### 2. 数据库需要有 role_type 字段

如果角色表 `sys_role` 没有 `role_type` 字段，需要先执行数据库迁移：

```bash
cd server/backend
alembic upgrade head
```

### 3. 前端路由需要更新

确保前端路由已经按照 `route-isolation-summary.md` 文档更新：
- 系统管理路由：`/admin/*`
- 业务功能路由：`/app/*`

## 🔧 故障排查

### 问题 1：登录后看不到任何菜单

**可能原因：**
- 角色没有关联菜单
- 角色的 `role_type` 字段值不正确

**解决方法：**
1. 检查角色是否关联了菜单（通过前端角色管理页面）
2. 检查数据库中 `sys_role.role_type` 字段值：
   ```sql
   SELECT id, name, role_type FROM sys_role;
   ```

### 问题 2：系统管理员能看到业务菜单

**可能原因：**
- 后端菜单过滤逻辑未更新

**解决方法：**
1. 检查 `server/backend/app/admin/service/menu_service.py` 中的菜单过滤逻辑
2. 确保实现了 `_filter_menus_by_role_type` 方法

### 问题 3：脚本执行失败

**可能原因：**
- 数据库连接失败
- 依赖未安装
- 数据库表结构不匹配

**解决方法：**
1. 检查数据库连接配置（`.env` 文件）
2. 安装项目依赖：`pip install -r requirements.txt`
3. 执行数据库迁移：`alembic upgrade head`
