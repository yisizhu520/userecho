# 反馈录入后的智能聚类与合并确认流程

> **日期**: 2025-12-30
> **需求**: 多入口反馈录入（截图/手动/Excel）后的聚类处理和主题合并确认

---

## 需求背景

用户反馈录入有三个入口：
1. **截图识别** - AI提取截图中的反馈内容
2. **手动录入** - 表单填写反馈
3. **批量导入** - Excel/CSV 批量上传

录入后需要：
- 处理 embedding 并触发聚类
- 单条反馈也要检测是否能合并到现有需求主题
- 用户确认合并到已有主题 or 创建新主题

---

## 设计决策

### 1. 聚类触发策略

| 录入方式 | 触发时机 | 触发类型 | 用户体验 |
|---------|---------|---------|---------|
| **手动录入** | 创建成功后 | 后台静默检测 | 不阻塞操作，列表页展示结果 |
| **截图识别** | 创建成功后 | 后台静默检测 | 不阻塞操作，列表页展示结果 |
| **批量导入** | 导入完成后 | 自动触发聚类 | 展示分组确认页面 |

### 2. 相似度阈值

- **默认值**: 70%
- **可配置**: 在聚类设置页面让用户调整
- **阈值影响**: 决定是否展示合并建议

## API 接口设计

### 1. 相似度检测接口

```typescript
/**
 * 检测单条反馈与现有主题的相似度
 * GET /api/v1/app/feedbacks/{feedbackId}/similar-topics
 */
interface SimilarTopicsResponse {
  feedback_id: string;
  status: 'pending' | 'processing' | 'completed' | 'failed';
  suggestions: Array<{
    topic_id: string;
    topic_title: string;
    similarity: number;        // 0-1
    feedback_count: number;
    category: string;
    match_reasons?: string[];  // 匹配原因说明
  }>;
  threshold_met: boolean;     // 是否达到用户配置的阈值
}
```

### 2. 批量聚类预览接口

```typescript
/**
 * 获取批量聚类的分组预览（不创建主题）
 * GET /api/v1/app/clustering/preview?feedback_ids=xxx,yyy,zzz
 */
interface BatchGroupsPreviewResponse {
  total_feedbacks: number;
  grouped_feedbacks: number;
  noise_count: number;
  ungrouped_count: number;
  groups: Array<{
    group_id: string;
    suggested_title: string;
    similarity: number;          // 组内平均相似度
    feedback_ids: string[];
    feedback_count: number;
    sample_contents: string[];   // 样本内容
    suggested_category: string;
  }>;
}
```

### 3. 批量确认创建主题

```typescript
/**
 * 根据分组预览批量创建主题
 * POST /api/v1/app/clustering/confirm-groups
 */
interface ConfirmGroupParams {
  group_id: string;
  title?: string;           // 可选：覆盖建议标题
  category?: string;        // 可选：覆盖建议分类
  description?: string;
  feedback_ids: string[];
}

interface BatchConfirmResponse {
  created_topics: number;
  topic_ids: string[];
  updated_feedbacks: number;
}
```

### 4. 合并反馈到主题

```typescript
/**
 * 将单条反馈合并到现有主题
 * PUT /api/v1/app/feedbacks/{feedbackId}/merge
 */
interface MergeToTopicParams {
  topic_id: string;
  confidence?: number;  // 记录用户操作时的相似度依据
}
```

### 5. 从建议创建新主题

```typescript
/**
 * 根据相似反馈创建新主题
 * POST /api/v1/app/topics/from-suggestions
 */
interface CreateTopicFromSuggestionsParams {
  feedback_ids: string[];
  title?: string;
  category?: string;
  description?: string;
}

interface CreateTopicFromSuggestionsResponse {
  topic: Topic;
  updated_feedbacks: number;
}
```

### 6. 聚类配置接口

```typescript
/**
 * 获取聚类配置
 * GET /api/v1/app/clustering/config
 */
interface ClusteringConfig {
  similarity_threshold: number;      // 默认 0.70
  batch_cluster_threshold: number;   // 默认 0.75
  auto_detect_on_create: boolean;    // 默认 true
  auto_cluster_on_import: boolean;   // 默认 true
}

/**
 * 更新聚类配置
 * PUT /api/v1/app/clustering/config
 */
interface UpdateClusteringConfigRequest {
  similarity_threshold?: number;
  batch_cluster_threshold?: number;
  auto_detect_on_create?: boolean;
  auto_cluster_on_import?: boolean;
}
```

## 用户操作流程总结

### 单条反馈流程
1. 用户填写反馈表单 / 上传截图
2. 点击「创建」
3. 显示成功提示，跳转到列表页
4. 后台异步检测相似主题
5. 列表页显示检测状态（检测中 → 有建议/无匹配）
6. 用户点击「查看建议」
7. 弹窗展示相似主题，用户选择：
   - 合并到现有主题
   - 创建新主题
   - 稍后处理

### 批量导入流程
1. 用户上传 Excel 文件
2. 显示导入进度
3. 导入完成后，自动触发聚类
4. 显示聚类进度
5. 聚类完成后，展示分组确认页面
6. 用户在分组确认页面：
   - 查看每个分组的详情
   - 编辑分组内容
   - 确认创建主题或忽略分组
7. 点击「全部确认」后批量创建主题

## 附录：相似度阈值建议

| 阈值 | 严格程度 | 推荐场景 | 预期效果 |
|-----|---------|---------|---------|
| 90% | 非常严格 | 对质量要求极高，宁可漏掉也不误合并 | 合并建议很少，但非常准确 |
| 80% | 严格 | 大部分场景的默认值 | 平衡准确率和召回率 |
| 70% | 中等 | 需要发现更多潜在关联，人工复核 | 会有一些误判，需要人工确认 |
| 60% | 宽松 | 探索性分析，寻找潜在关联 | 建议较多，需要大量人工筛选 |

**推荐默认值**: 70%（合并建议） / 75%（批量聚类）
