# 中国版 Canny (userecho) 产品路线图

> **目标**: 将 userecho 从内部管理工具转型为适配中国市场的「Canny」,提供公开反馈门户、路线图和更新日志。

---

## 一、产品定位与差异化策略

### 核心定位
**面向中国独立开发者和中小企业的 AI 驱动反馈管理平台**

### 与 Canny 的差异化
1. **AI 优先**: 深度 AI 聚类 + 截图智能识别(微信/小红书/App Store)
2. **中国生态**: 微信登录、飞书/钉钉集成、微信小程序
3. **低门槛**: Excel 批量导入、快速上手
4. **渐进式公开**: 先做内部工具,再逐步开放公开门户

---

## 二、分阶段路线图

### 📍 Phase 1: 透明化 (MVP - 第 1-3 周)

**目标**: 快速上线只读公开页面,建立用户信任,完成反馈闭环

#### 核心功能
1. **公开路线图** (3-5 天)
   - Kanban 看板视图:「计划中」「进行中」「已完成」
   - 按优先级、时间排序
   - 支持筛选(分类、状态)
   - SEO 优化

2. **更新日志 (Changelog)** (7-10 天)
   - 时间线视图展示产品更新
   - Markdown 富文本编辑
   - 关联已完成的 Topic
   - RSS 订阅支持

3. **基础设施** (5-7 天)
   - 新建 `web-portal` 前端应用
   - 后端 Public API 路由
   - 数据模型扩展(Topic.is_public, Changelog 表)

#### 验收标准
- [ ] 用户可访问 `yourdomain.com/roadmap` 查看公开路线图
- [ ] 用户可访问 `yourdomain.com/changelog` 查看更新历史
- [ ] Admin 可在后台标记 Topic 为「公开」
- [ ] Admin 可发布 Changelog 并关联 Topic

---

### 📍 Phase 2: 社区化 (第 4-6 周)

**目标**: 启用用户互动,建立反馈社区

#### 核心功能
1. **公开反馈看板** (7-10 天)
   - 用户可浏览所有公开 Topic
   - 搜索、筛选、排序
   - 投票功能(点赞 Topic)
   - 热门需求自动置顶

2. **用户认证系统** (10-15 天)
   - **微信扫码登录**(优先级最高)
   - 邮箱 Magic Link 登录
   - 创建 `EndUser` 表(区分 Admin)
   - JWT Session 管理

3. **评论与讨论** (7-10 天)
   - Topic 下的评论线程
   - 公开评论 + 内部评论(Admin Only)
   - @提醒功能
   - 评论通知

4. **用户提交反馈** (5-7 天)
   - 简化的反馈提交表单
   - AI 自动去重(检测相似 Topic)
   - 自动分类和打标签

#### 验收标准
- [ ] 用户可通过微信扫码登录
- [ ] 用户可对 Topic 投票
- [ ] 用户可提交新反馈
- [ ] 用户可在 Topic 下评论讨论

---

### 📍 Phase 3: 生态化 (第 7+ 周)

**目标**: 深度融入中国开发者生态

#### 核心功能
1. **微信生态集成**
   - 微信小程序(可选,根据需求评估)
   - 微信公众号菜单接入
   - 微信消息通知

2. **企业协作工具集成**
   - 飞书通知(Webhook)
   - 钉钉通知(Webhook)
   - Slack 通知(兼容海外用户)

3. **截图解析 2.0**
   - 优化现有截图识别准确率
   - 支持用户在公开门户上传截图
   - 批量处理优化

4. **用户分群与分析**
   - 自定义字段(客户类型、MRR 等)
   - 按用户分群筛选反馈
   - 数据看板(反馈趋势、用户活跃度)

#### 验收标准
- [ ] 新 Topic 自动推送到飞书/钉钉
- [ ] 用户可在公开门户上传截图反馈
- [ ] Admin 可按客户分群查看反馈

## 四、核心假设与风险

### 核心假设
1. ✅ **AI 聚类质量**是核心竞争力(已验证)
2. ⚠️ **截图识别**是中国市场刚需(待验证)
3. ⚠️ **公开门户**能提升用户参与度(待验证)
4. ⚠️ 用户愿意通过**微信登录**参与反馈(待验证)

### 主要风险
1. **公开门户冷启动**: 初期用户少,社区不活跃
   - **缓解**: Phase 1 先做只读页面,降低期望
2. **微信登录审核**: 微信开放平台审核周期长
   - **缓解**: 同步开发邮箱登录作为备选
3. **功能过载**: 对标 Canny 导致开发周期过长
   - **缓解**: 聚焦 Phase 1 MVP,快速验证
