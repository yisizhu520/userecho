# 回响（UserEcho）演示数据设计

> **目标**: 设计与"回响"产品高度相关的演示数据，展示产品核心功能和价值主张

---

## 一、产品核心定位

**回响** 是面向中国独立开发者和中小企业的 **AI 驱动用户反馈分析平台**

### 核心功能
1. **智能聚类** - AI 自动将相似反馈聚合为议题（Topic）
2. **截图识别** - 识别微信/小红书/App Store 截图中的文本
3. **优先级评分** - 基于商业价值、影响范围、紧急度智能打分
4. **状态追踪** - 从「新提交」→「评审中」→「已批准」→「开发中」→「已发布」全流程管理
5. **公开路线图** - 向用户透明展示产品开发计划（Phase 1 核心功能）
6. **更新日志** - 记录产品迭代历史（Phase 1 核心功能）

### 目标用户
- **SaaS 创业公司** - 需要快速响应用户反馈
- **独立开发者** - 个人产品，需要低成本工具
- **中小企业产品团队** - PM + CS 协作场景

## 四、用户反馈内容设计

### 设计原则
1. **真实性** - 基于回响自身的真实问题和需求
2. **多样性** - 覆盖不同功能模块和用户场景
3. **可演示性** - 能展示 AI 聚类、优先级评分等核心功能

### 反馈分类

#### A. AI 聚类与分析相关
```
1. "AI 聚类效果很好，但希望能手动调整聚类结果"
2. "能否支持自定义聚类规则？比如强制将某些关键词归为一类"
3. "聚类后的议题名称有时不够准确，希望能批量重命名"
4. "希望能看到每个议题的聚类置信度分数"
5. "能否支持多语言聚类？我们有英文和中文混合的反馈"
```

#### B. 截图识别与图片处理
```
1. "微信截图识别准确率很高，但小红书截图偶尔识别不出来"
2. "能否支持批量上传截图？现在只能一张张传"
3. "截图识别后希望能自动提取关键信息（如版本号、错误码）"
4. "App Store 评论截图能否直接识别星级和用户昵称？"
5. "上传的截图希望能自动压缩，现在图片太大加载慢"
```

#### C. 优先级评分与决策
```
1. "优先级评分公式能否自定义？我们更看重商业价值而不是影响范围"
2. "希望能批量修改议题的优先级"
3. "能否增加'开发成本'字段？现在只能手动估算"
4. "优先级排序希望支持拖拽手动调整"
5. "能否给出 AI 推荐的优先级评分？"
```

#### D. 状态追踪与协作
```
1. "状态流转能否发送通知？比如从'评审中'变为'已批准'时通知相关人"
2. "希望在议题下方增加评论功能，方便团队讨论"
3. "能否支持 @提醒同事？"
4. "状态变更历史希望能看到具体是谁在什么时间操作的"
5. "能否自定义状态流？我们的流程是：提交→待评估→开发中→测试中→已上线"
```

#### E. 公开路线图与透明化（Phase 1 核心）
```
1. "希望能有公开路线图功能，让用户看到我们在做什么，提升信任度"
2. "能否让用户对路线图的议题投票？"
3. "希望路线图有看板视图和时间线视图两种展示方式"
4. "能否支持将路线图嵌入到我们的官网？"
5. "希望路线图能显示预计完成时间"
```

#### F. 更新日志（Changelog）
```
1. "希望能有更新日志功能，支持 Markdown 编辑和图片上传"
2. "能否自动将'已完成'状态的议题生成 Changelog 草稿？"
3. "Changelog 希望支持 RSS 订阅"
4. "能否在 Changelog 中展示关联的反馈数量？让用户知道这个功能有多少人想要"
5. "希望 Changelog 能自动推送到微信公众号"
```

#### G. 数据导入与集成
```
1. "Excel 导入很方便，但能否支持 CSV 格式？"
2. "希望能对接飞书多维表格，自动同步反馈"
3. "能否提供 API 接口？我们想从客服系统自动推送反馈"
4. "批量导入时希望能预览数据，确认无误后再提交"
5. "能否支持从 Canny / ProductBoard 导入历史数据？"
```

#### H. 客户管理与分群
```
1. "客户列表希望能按 MRR 排序，优先看到大客户的反馈"
2. "能否给客户打标签？比如'重点客户''流失风险'等"
3: "希望能按客户类型筛选反馈（VIP / Paid / Free）"
4. "能否自动关联客户的历史反馈？方便了解客户的完整需求"
5. "大客户的反馈希望能自动标记为高优先级"
```

#### I. 通知与提醒
```
1. "新反馈提交时希望能发送飞书通知"
2. "能否支持钉钉 Webhook？"
3. "邮件通知的内容希望能自定义模板"
4. "能否设置通知规则？比如只有紧急反馈才通知我"
5. "希望能在微信接收通知推送"
```

#### J. 数据分析与可视化
```
1. "希望能看到反馈趋势图（按周/月统计）"
2. "能否增加仪表盘，展示关键指标（反馈总数、处理率、平均响应时间）？"
3. "希望能导出数据报表（Excel），给老板汇报用"
4. "能否按来源渠道统计反馈（微信、邮件、CRM）？"
5. "希望能看到每个议题的影响用户数"
```

#### K. 移动端与体验优化
```
1. "希望有移动端 App，方便随时查看反馈"
2: "暗色模式什么时候能上线？晚上用太刺眼了"
3. "列表页加载速度有点慢，数据多的时候卡顿"
4. "搜索功能希望支持模糊匹配"
5. "能否记住我的筛选条件？每次都要重新选很麻烦"
```

#### L. 权限与安全
```
1. "能否设置只读权限？我们的 CS 团队只需要查看，不需要编辑"
2. "希望能看到操作日志（谁删除了哪条反馈）"
3. "能否支持 SSO 单点登录？"
4. "敏感客户的反馈希望能设置为私密，只有特定人员可见"
5. "数据备份功能什么时候有？"
```

### 状态分布
- **新提交** (New): 5 个
- **评审中** (Review): 4 个
- **已批准** (Approved): 3 个
- **开发中** (In Progress): 3 个

### 优先级说明
- **90+** - 紧急且重要（如性能问题）
- **80-89** - 高优先级核心功能
- **70-79** - 中优先级改进
- **60-69** - 低优先级优化
- **<60** - 观望或延后

## 七、时间分布策略

### 反馈提交时间
- **最近 7 天**: 30% 的反馈
- **8-14 天前**: 25% 的反馈
- **15-30 天前**: 30% 的反馈
- **30 天以上**: 15% 的反馈

### 议题创建时间
- **最近 3 天**: 「新提交」状态的议题
- **1-2 周前**: 「评审中」和「已批准」的议题
- **2-4 周前**: 「开发中」的议题

## 十、关键展示场景

### 场景 1: AI 聚类效果
**演示点**: 展示相似反馈如何被 AI 自动归类为议题
- 议题「议题评论与讨论」包含 3 条相似反馈
- 议题「飞书/钉钉通知」包含 6 条反馈（来自不同客户）

### 场景 2: 优先级智能评分
**演示点**: 展示不同议题的优先级评分差异
- 「列表性能优化」(91.2) vs 「暗色模式」(55.4)
- 说明评分逻辑：商业价值 × 影响范围 × 紧急度

### 场景 3: 客户价值区分
**演示点**: VIP 客户的反馈权重更高
- Wolai（VIP）提的需求优先级自动 +20%
- 试用用户的需求相对靠后

### 场景 4: 公开路线图
**演示点**: Phase 1 核心功能
- 展示「已批准」和「开发中」的议题在路线图看板
- Changelog 记录已完成的功能
