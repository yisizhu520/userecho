> **作者**：Linus Torvalds 视角  
> **日期**：2025-12-23  
> **配套**：clustering-refactor-plan.md（技术方案）  
> **原则**：透明化 > 智能化 | 掌控感 > 自动化

---

## 🎯 设计哲学

### 核心原则

> "给用户一个明确的心理预期，而不是黑盒魔法"

**问题**：传统 AI 产品的通病
- 用户上传数据 → 黑盒处理 → 突然出现结果
- 不知道 AI 在干什么
- 不知道结果好不好
- 不知道能不能改

**我们的方案**：
- ✅ **实时反馈**：每一步都告诉用户"发生了什么"
- ✅ **质量透明**：AI 置信度、聚类质量都可见
- ✅ **人类最终决策**：AI 只提供建议，PM 决定采纳或拒绝
- ✅ **可回退**：所有操作都可撤销

#### 1.2 状态图标与含义

| 图标 | 状态 | 含义 | 用户操作 |
|------|------|------|---------|
| ✅ | 已分类 | 已归入某个主题 | 可重新分类 |
| 🔄 | 处理中 | AI 正在分析 | 可取消 |
| ⏳ | 待处理 | 等待聚类 | 可手动分配或触发聚类 |
| 🎲 | 噪声点 | AI 无法聚类 | 建议手动归类 |
| ❌ | 失败 | 处理出错 | 可重试 |

      const intervalId = setInterval(async () => {
        const status = await api.getTaskStatus(taskId)
        this.tasks[taskId] = status
        
        if (status.status === 'completed' || status.status === 'failed') {

### 2. WebSocket 优化（可选）

**场景**：大量反馈导入，实时推送进度

```typescript
// composables/useClusteringWS.ts
export function useClusteringWebSocket(taskId: string) {
  const ws = new WebSocket(`ws://localhost:8000/ws/clustering/${taskId}`)
  
  ws.onmessage = (event) => {
    const progress = JSON.parse(event.data)
    // 更新进度条
    updateProgress(progress)
  }
  
  ws.onerror = () => {
    // 降级到轮询
    fallbackToPolling(taskId)
  }
  
  return { ws }
}
```

## 🎯 总结

### 设计亮点

1. **透明化**：每一步都告诉用户"发生了什么"
2. **可控性**：AI 只提供建议，人类做最终决策
3. **渐进式**：从简单的轮询开始，后期可升级 WebSocket
4. **容错性**：所有操作都可撤销，低质量结果会被过滤

### 实施优先级

**P0（MVP 必须）**：
- 反馈列表状态展示
- 导入进度可视化（轮询）
- AI 发现中心基础功能

**P1（短期优化）**：
- 拖拽归类
- 质量评分展示
- 合并建议

**P2（长期优化）**：
- WebSocket 实时推送
- 趋势分析图表
- 邮件通知
